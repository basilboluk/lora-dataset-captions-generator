# LoRA Dataset Captions Generator

A desktop tool to streamline the creation of detailed captions for images and video frames when building datasets for LoRA training, tested with Ostris AI Toolkit and ComfyUI

## About

Preparing hundreds or thousands of `.txt` captions for images and video frames sounds simple, but quickly becomes exhausting manual work that kills motivation on large LoRA projects. This tool turns that process into a structured, form-based workflow so you can focus on dataset quality instead of repetitive typing.

The workflow comes from engineering projects where dozens of 3D models and assemblies are exported from SolidWorks into KeyShot Studio. Instead of manually rebuilding complex scenes in Houdini, Blender, Cinema 4D, etc., it is often more efficient to rely on diffusion AI models — as long as you maintain **consistency** across your dataset.

Text-to-image models cannot create what they have never seen, and image-to-image/video workflows suffer when the reference dataset is too small or poorly captioned. In practice, LoRA is one of the most reliable ways to preserve design consistency across many shots.

A simple count shows the scale: if you need around 26 camera angles per model, and each model has options, modifications, multi-level details, isolated parts and dynamic states, 30 base models can easily become 780+ images — all requiring clean, consistent captions. Only then can the diffusion model’s encoder correctly learn how to “build” the image.

In one of my use cases, two projects required separate reference folders such as:

1. `1_drone_entire_object`
    
2. `2_headset_entire_object`
    
3. `3_large_nodes`
    
4. `4_isolated_options`
    
5. `5_isolated_details`
    

Each folder contained rendered outputs from KeyShot Studio, and for every render I created a `.txt` caption with the same base name. Ostris AI Toolkit then automatically assembled the dataset and correctly attached captions to each image.

You can customize this tool to match your own dataset structure, naming, and property sets.

## Features

- Batch-oriented caption generation for LoRA training datasets.
    
- Form-based UI for consistent, structured captions across many images.
    
- Category-based saving: each caption `.txt` is stored in a folder that corresponds to a logical dataset category.
    
- Automatic image/caption name pairing: captions use the same base name as the image file, making dataset assembly straightforward in Ostris AI Toolkit, ComfyUI, and similar tools.
    
- JSON-based schema (`dataset_descriptions.json`) that defines categories, fields, and allowed values, so you can adapt the tool to your own domains.

## Usage

### Requirements

- Python environment.
    
- Microsoft Visual Studio or Visual Studio Code as the IDE to run the Python script (the examples assume Visual Studio).
    

### Running the caption generator

1. Open the project in Visual Studio (solution `DroneHeadsetDataset.sln`, or your renamed project).
    
2. Open `json_handler.py`.
    
3. Run the script via the **Start** button in the IDE toolbar.
    
4. Two windows appear:
    
    - A console/debug log window (can be closed if not needed).
        
    - The main form window: **“Caption generator by categories”**.
        

The script automatically loads `dataset_descriptions.json`, which defines the form schema and available categories.

## Workflow

1. JSON schema is loaded automatically at startup.
    
    - If you work with multiple data structures, use **“Load template JSON”** to switch schemas.
        
2. Select a **category** from the dropdown at the top.
    
    - Generated `.txt` files will be saved into a folder with this category name.
        
3. Fill in the form fields:
    
    - Use dropdowns for predefined properties.
        
    - Use text inputs where free-form values are needed.
        
    - Skip `image_name` — it is generated automatically to match the `.txt` name.
        
4. Click **“Generate .txt”**:
    
    - The folder with the category name is created automatically (if not present).
        
    - A caption file is generated and saved.
        
5. File naming:
    
    - The file name is generated automatically as `CategoryName_#`, where `#` is an incremental identifier.
        
    - Files are saved under `/dataset/yourcategoryname`.
        

Use these `.txt` captions alongside image files in your LoRA training pipeline (e.g. Ostris AI Toolkit datasets, ComfyUI LoRA workflows).

## Customization and Modification

You can adapt the tool to any domain by editing the JSON schema and the generator logic.

### 1. Adjusting the data schema

- Edit `dataset_descriptions.json` to:
    
    - Add/remove categories.
        
    - Change property names and groups.
        
    - Define single-select lists, multi-select lists, or free text fields.
        

This file controls what appears in the form and how captions are structured logically.

### 2. Updating the script logic

- In `json_handler.py`:
    
    - Update `"categories"` in `initial_data` to reflect your category list.
        
    - Adjust `self.schemas_by_category` to define which fields belong to each category and how they behave (single select, multi select, text input).
        
    - Update `"templates"` in `def get_template(self, cat):` to control the final `.txt` caption structure per category.
        

This lets you tailor captions to any specific LoRA training scenario: engineering models, characters, props, environments, etc.
